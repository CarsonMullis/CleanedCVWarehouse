About the Project

The goal of this project was to develop a Digital Twin, a system that keeps track of of assets in the warehouse and allows users to inspect assets at a distance.


We built a warehouse intelligence system that uses four ceiling-mounted corner cameras to monitor and interpret activity within a live warehouse environment. Our system leverages YOLO-based object detection to identify, track, and manage warehouse assets in real time, including robots, pallets, crates, and workstations.


To create spatial awareness of the warehouse, we developed multiple 3D environment reconstruction approaches. One environment was built using Three.js, where a predefined warehouse boundary and captured images are used to position detected objects within an interactive 3D scene. In parallel, we implemented a second reconstruction pipeline that uses video input and AI-based processing to generate a Gaussian Splat representation of the warehouse, allowing users to visualize the environment in a detailed and immersive 3D format.


We also developed interactive 3D object modeling capabilities that reconstruct warehouse objects from photographs, enabling users to inspect and interact with generated models directly within the system. Detection and tracking results produced by our computer vision pipeline can be saved for later analysis, providing persistent records of warehouse activity and object placement over time.


Beyond visualization and detection, we integrated direct interaction with the AVA mobile robot through a browser-based interface. Using API-driven controls, operators can communicate with and control the robot directly from the web application, allowing the system to bridge real-time warehouse perception with robotic interaction and navigation.

For Those Who Come After…

To anyone who gets the reference, gold star for you.


This area is intended for those who did not originally start the project but are looking to improve or expand upon it, whether as a capstone project of their own or simply to experiment with the system.


This project is composed of three subprojects: Object Detection with AVA, the 3D Environment, and the 3D Modeling of warehouse objects. Each component came with its own challenges and frustrations.


For the Object Detection system, one of the initial challenges was finding a suitable vision model and improving its ability to accurately capture and detect objects through the warehouse cameras. Toward the end of development, the focus shifted to integrating the AVA robot and enabling it to navigate through simple interactions, such as clicking locations on the live camera display feed.


Because this project combines three distinct subprojects, it can become easy for team members to work in silos and lose awareness of the progress being made in other areas. It is important that everyone remains informed about the current state of each subsystem, as this improves overall cohesion and strengthens each member’s ability to discuss and contribute to the project.


Those looking to improve the project may explore alternative object detection models, further implementation of Gaussian Splats, improvements to AVA’s ability to capture photos and autonomously explore the warehouse for 3D generation or object detection, and expanded 3D object modeling for applications such as damage detection and warehouse asset storage.


As time goes on, I hope this area becomes filled with stories, notes, and lessons from the people who worked on the project — including the challenges they faced, the successes they achieved, and the warnings or advice they wish to pass on to future teams.
